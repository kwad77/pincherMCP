`pincher bench` now measures the `trace` row instead of reporting a flat 0% savings ([#1809](https://github.com/kwad77/pincher/issues/1809)). `runBenchSuite` passed `nil` edgeKinds to `TraceViaCTEScoped`, which errors on empty edgeKinds; the swallowed error made every trace call return nil, so the trace row showed actual=1 / baseline=0 / savings=0% on every run. bench now passes the CALLS-family edge kinds the MCP trace tool defaults to, and `benchSampleSymbolsWithEdges` requires an outbound CALLS edge (bench traces outbound — an inbound-only leaf sample would still floor the row). On the pincher-repo index the trace row went from 0% to 99.9% measured savings.
