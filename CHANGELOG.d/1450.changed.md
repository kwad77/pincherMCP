Swift extractor promoted from confidence **0.70 → 0.85** (stable-regex tier, joining TS/TSX/Rust/Java). `classRE` now covers `class` / `struct` / `actor` (modern Swift's three nominal type kinds); dedicated `enumRE` catches enum declarations; `funcRE` captures `func` / `init` / `subscript` declarations with `@AttributeName` and `@AttributeName(args)` prefix tolerance (e.g. `@MainActor`, `@objc`, `@discardableResult`, `@available`), `class func` static methods, `mutating` / `nonmutating` / `consuming` / `borrowing` modifiers, `override` / `required` / `convenience` / `final` / `lazy` modifiers, and `fileprivate` access level. `scopeRE` (extensions) tolerates `where`-clause constraints. Generic signatures (`func foo<T>`, `func foo<T: Comparable>`) work — the `name` capture stops at `<`. Pre-fix every struct, enum, actor, and constructor in a Swift codebase was invisible to `dead_code`/`search`/`trace`; a typical SwiftUI codebase (50% structs + 30% enums + every type's `init`) was effectively under-extracted. AST-tier extractor (subprocess to `swift-syntax` mirroring Python's #856 pattern) tracked separately for v0.85+. (#1450)
