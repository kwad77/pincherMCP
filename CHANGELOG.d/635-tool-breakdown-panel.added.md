Dashboard panel: **Tool Call Breakdown (last 7 days)**. First v0.67 panel rendering data from the schema v27 `session_tool_calls` event log (#635 substrate landed in v0.64; this is the panel layer). Shows one row per tool with call count, average tokens used, cumulative tokens saved, and average savings percentage, plus a 7-day total footer. Admin-shape tools without a Read/Grep baseline (architecture/list/schema) show `—` in the saved% column rather than 0% so users don't misread them as "no savings." Backed by new `/v1/tool-call-stats?window_seconds=…&limit=…` HTTP endpoint and `ToolCallStatsByTool` DB query — both reader-routed; GROUP BY tool with `AVG()` over the trailing window. Empty-store path returns `tallies:[]` not `null` (the #330 invariant the dashboard JS relies on). NULL `tokens_saved_pct` rows excluded from the average so admin tools don't drag the search/symbol numbers down. ([#635](https://github.com/kwad77/pincher/issues/635))
