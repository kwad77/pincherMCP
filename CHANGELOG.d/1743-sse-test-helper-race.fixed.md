Fixed a data race in the `readSSEFrames` test helper, surfaced by the `-race` CI job (#1743 v0.89). The helper spawns a goroutine that scans an SSE response and appends to a shared `frames` slice; on its timeout path it returned that slice while the goroutine was still appending — racing the caller's reads. On timeout it now closes the response body (unblocking the scanner so the goroutine exits) and waits for the goroutine before returning. Test-only race — `readSSEFrames` is a test helper; production SSE code is unaffected. Third race cleared from the `-race` job's findings (9 → 3 → 0).
