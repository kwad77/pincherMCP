The HTTP dashboard's Overview "Session" card now shows the agent's real token savings instead of 0 ([#1815](https://github.com/kwad77/pincher/issues/1815)). `GET /v1/stats` picked the newest session row by `last_seen`, but a standalone `pincher web` dashboard process polls itself every 30s and keeps its own poll-noise session (0 saved) the freshest — so the card showed dashboard noise, never the agent's MCP session. In `--no-stdio` mode `/v1/stats` now picks the newest session with an empty `http_url` (the agent's pure-MCP stdio session). Separately, the dashboard's ~15 inline `style=` attributes — blocked on every load by CSP `style-src 'self'` ([#1814](https://github.com/kwad77/pincher/issues/1814)) — are gone: static styling is class-based, dynamic bar sizing is applied via CSSOM after render. The strict CSP is unchanged.
