TypeScript extractor now extracts abstract methods and indented type declarations (#1761 v0.89). `tsRE.methodRE`'s modifier alternation listed public / private / protected / readonly / static / async but not `abstract`, so an `abstract run(): void;` declaration had `abstract` captured as the method name and then failed the `(` anchor — every abstract method on every abstract class was silently dropped. `methodRE` now accepts `abstract` (sharing the optional group with `static`, since the two are mutually exclusive in TS). Separately, `classRE` / `interfaceRE` / `enumRE` anchored at column 0 with a bare `^` (only `methodRE` carried `^\s*`), so a class / interface / enum indented inside a `namespace` block — or otherwise nested — was never extracted; all three now lead with `^\s*`. The related gap where a `namespace` block is not a scope (so its `function` members orphan) needs a framework change and is tracked in #1762. Found dogfooding the regex-tier extractor sweep.
