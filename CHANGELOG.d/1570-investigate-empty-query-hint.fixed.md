**investigate_failure: empty-suspects next_steps no longer emit `{"query":""}` (#1570).** When the trace parser captured file paths but no identifier-shape names (e.g. a Python traceback whose paths matched but whose name regex didn't), the empty-suspects branch's next-step `{"tool":"search","args":{"query":""}}` broke the failure-as-pedagogy chain — handleSearch rejects empty queries. Now the search hint uses the first frame name if available, falls back to the file basename (extension stripped) when only file paths were parsed, or is omitted entirely if neither — `doctor` always remains. Regression test pins the contract: no `search` next_step in this surface has an empty-string query. Part of composite-hardening umbrella #1582.
