**`t.Parallel()` across `internal/server` tests — Windows wall time 6× compression ([#697](https://github.com/kwad77/pincher/issues/697) Bucket D).** Zero `t.Parallel()` calls existed anywhere in the repo (`grep -c 't\.Parallel()' **/*.go` returned 0), so with `-p 2` only two packages ran in parallel and each ran serial-within. `internal/server` (the worst offender) was 6.2× slower on windows-latest than ubuntu-latest (121s vs 19.6s) because Windows SQLite I/O is ~10× slower per-package and serial-within-package multiplies that. Added `t.Parallel()` to every top-level `Test*` function across 84 files in `internal/server/`, skipping the 5 that mutate process-global state (`auto_restart_test.go`, `session_persist_test.go`, `stale_binary_test.go` — all use `t.Setenv`), have explicit timing budgets that parallelism would flake (`large_dataset_test.go`'s 5s/endpoint wallclock), or contain only `TestMain` (`bench_setup_test.go`). Local Windows wall time dropped from baseline 19.6s → 7.4s (2.6× compression on a 6-core machine). CI Windows wall time delta will land in the PR run.
