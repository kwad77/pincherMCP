**Per-tool latency budget (FILE-C, #1522).** `testdata/bench/per-tool-latency-budget.json` pins per-tool p50/p99 targets for 18 advertised tools (`search`, `symbol`, `context_for_task`, `investigate_failure`, `trace`, `query`, etc.). `scripts/per-tool-latency.sh` indexes a pinned corpus, starts the HTTP gateway, runs 100 iterations per tool, computes p50/p99, and compares p99 against `budget × 1.20`. `.github/workflows/per-tool-latency.yml` runs weekly (Thursday 12:00 UTC) + on `workflow_dispatch`. **Phase 1 (v0.91-v0.99): advisory** (`continue-on-error: true`, script exits zero on threshold breach — failures surface as `::warning::` log lines). **Phase 2 (v1.0+): required** per FILE-C acceptance criterion — one-line edits in workflow + script flip the gate to release-blocker. Methodology + when-to-update-budget documented in [docs/methodology/per-tool-latency.md](docs/methodology/per-tool-latency.md). Closes the single-tool-regression-hides-in-aggregate gap that `make corpus-bench` can't catch.
