`pincher doctor` (CLI) and the MCP `doctor` tool accept a `project` arg / `--project` flag that scopes the projects + extraction_failures sections to projects whose name or id contains the case-insensitive substring (same shape as `pincher project rm` and `pincher verify --project`). Database-level advisories (large-DB, ghost-project, nested-project, WAL bloat, branch drift) remain project-wide — they're store-wide signals. Fixes the dogfood friction of doctor returning every project + every project's failures when the user only cares about one. (#1401)
