`pincher init --target=antigravity` now writes a path Google Antigravity actually reads (#1765 v0.89). The target shipped (#1385) writing `./.antigravity/rules.md` — a guessed `.<tool>/rules.md` convention. Antigravity has no `.antigravity/` directory: per Google's own Antigravity codelab and rules documentation it reads workspace rules from the `.agents/` directory (`.agents/rules/` holds supplementary rule files, alongside `.agents/workflows/` and `.agents/skills/`), project-root `GEMINI.md` / `AGENTS.md`, and global `~/.gemini/GEMINI.md`. So `pincher init --target=antigravity` produced a file Antigravity silently ignored. The target now writes `./.agents/rules/pincher.md` — a dedicated pincher rules file inside Antigravity's workspace-supplements directory, mirroring the `cursor` target's `.cursor/rules/pincher.mdc` shape (idempotent re-runs, never clobbers the user's own `GEMINI.md` / `AGENTS.md`). `DetectFn` keys on the `.agents/` directory; `--global` is rejected with a pointer to the `gemini` target, which owns `~/.gemini/GEMINI.md`. Found verifying pincher's host integrations after Antigravity's latest release.
