The #1772 incremental-resolve self-heal now also fires for a project that never goes idle (v0.89, #1772 follow-up). The #1774 watcher self-heal re-runs a full resolve-only pass on an active→idle transition to restore CALLS edges a scoped incremental resolve dropped — but it only fired on that transition. A continuously-active project (a busy dogfood/dev session with edits every watcher tick) never reaches the idle branch, so the edge drift accumulated unbounded for the whole session (observed live: 14795 → 14261 CALLS edges over a session before a force pass restored them). The watcher now counts incremental reindexes per project and forces the resolve-only self-heal once the count crosses a threshold even while the project is still active, bounding the drift window regardless of whether the project ever settles. Root cause of the scoped-resolve drop itself stays open under #1772.
