**Cypher test-invariant tightening ([#672](https://github.com/kwad77/pincher/issues/672) workstream 4 v0.79 hardening):** four cypher tests (`TestParseProps_InlineNodeFilter`, `TestParseProps_EmptyBraces`, `TestParseProps_InlineToNodeFilter`, `TestRunBFS_WhereFilter`) had defensive `if r.Total == 0 { t.Skip("may not be supported") }` branches that masked silent feature regressions — the test would PASS-as-skipped if the inline-props or BFS-WHERE filter returned zero rows, even when the corpus clearly had matching symbols. Swapped each skip for `t.Fatal` so the tests assert what they claim: \"on a corpus with one matching symbol, the filter returns that symbol.\" Same silent-confidently-wrong shape as the v0.59 family (#1121); now closed in cypher tests as well as runtime paths.
