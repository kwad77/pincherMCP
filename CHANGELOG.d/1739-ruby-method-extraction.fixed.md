Ruby extractor fixes (#1739 v0.89). `def self.create` (a Ruby class method) was extracted with the name `self` — `funcRE` captured the receiver, not the method name, so every `def self.X` in a file collapsed to `self` and collided; `funcRE` now skips an optional `self.` / `Receiver.` prefix. `classRE` anchored on bare `^class`, so a class indented inside a module — idiomatic Ruby — was never extracted; it now uses `^\s*` and also matches `module` (Ruby's primary namespace container, previously not extracted at all). Found dogfooding the extractor across languages the #1389 cross-language sweep did not cover.
