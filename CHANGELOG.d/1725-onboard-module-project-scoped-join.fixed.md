`onboard_module` no longer multiplies boundary edges across indexed projects (#1725 v0.89). Its edge query resolved from/to symbol names with `LEFT JOIN symbols fs ON fs.id = e.from_id` — joining on `id` alone. The `symbols` primary key is `(project_id, id)`, so the same symbol id exists in every project that indexed it (forks, the nested-project registrations `pincher doctor` warns about); the unscoped join matched all of them, multiplying every `external_dependencies` / `external_consumers` entry by the count of projects sharing the id. Dogfooding `onboard_module` against a multi-project install showed each dependency listed 2× (and up to 4× when both join sides collide). The joins are now scoped to `e.project_id` on both sides — same #1232-class cross-project-leak fix as the `search`/`symbol`/`trace` session-project divergence (#1408).
