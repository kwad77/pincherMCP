C# regex extractor no longer lifts control-flow keywords to phantom Method symbols (#1693 v0.89). The #1389 cross-language false-positive sweep against 27 real multi-language dogfood projects surfaced 98 `qualified_name_collision` extraction failures, dominated by C / C++ / C#. The C# class: the `funcRE` shape `(?:[\w...]+\s+)+ name \(` matches `else if (cond)` — `else` is consumed as a return-type token, `if` lands in the name capture, `(` follows — so every C# file with an `else if` emitted a phantom `if` Method, and repeated ones collided on the qualified name. New `dropCSharpKeywordFalsePositives` filters Function/Method symbols whose name is exactly a C# reserved control-flow keyword (`if`/`else`/`for`/`foreach`/`while`/`do`/`switch`/`case`/`lock`/`using`/`fixed`/`try`/`catch`/etc. — 22 keywords the compiler rejects as identifiers). Mirrors C's `dropCKeywordFalsePositives` (#1148). Exact-name match — a method named `ifMatches` or `forEach` survives. The C/C++ `_API`-macro class and the C# type-reference class from the same sweep are tracked in #1693 as follow-ups.
