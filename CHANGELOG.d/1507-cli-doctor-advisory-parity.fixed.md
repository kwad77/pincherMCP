**CLI ↔ MCP doctor advisory parity restored ([#1507](https://github.com/kwad77/pincher/issues/1507) v0.83):** three advisories that previously shipped only on the MCP doctor side — `ghostProjectAdvisory` (#815/#1009 — projects with substantial symbols but vanishingly few edges, the "ghost-extraction signature" where extraction ran but resolver phase produced no graph), `walBloatAdvisory` (#1206 — WAL file past the 256 MB journal_size_limit soft cap, fires above 512 MiB absolute OR >10% of a 100+ MiB DB), and `nestedProjectAdvisory` (#1209 — inner project indexed under an outer project's path, doubling DB load) — are now mirrored to `cmd/pinch/doctor.go` so `pincher doctor --json` surfaces the same conditions an agent would see via the MCP `doctor` tool. CLAUDE.md's bounded-duplication convention already declared this requirement; pre-fix the policy was documented but the implementation had drifted. Each port is a byte-for-byte mirror with a `mirrors internal/server/admin.go's copy` comment + matching thresholds (ghost-project: 1000-symbol floor, 0.001 ratio floor; wal-bloat: 512 MiB absolute, 10% / 100 MiB ratio; nested-project: 3-pair cap, path-separator-after-parent guard). 14-test mirror suite pins each advisory's positive/negative/control cases so future drift between the two copies can be diffed against the matching `internal/server/doctor_*_advisory_test.go` files at PR review time.
