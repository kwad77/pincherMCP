The symbol graph no longer degrades over a working session ([#1772](https://github.com/kwad77/pincher/issues/1772)). The CALLS resolve pass replaced its edge set with a `DELETE` commit followed by a separate `INSERT` commit; between the two, the committed DB held zero `resolve_pass` CALLS edges, so any concurrent MCP read (`trace` / `query` / `changes` / `dead_code`) landing in a watcher's resolve window saw cross-file CALLS edges vanish — `dead_code` false-positived heavily-called functions, `trace` reported "no callers". The new `ReplaceEdgesByKindSource` does the delete and the re-insert in one transaction; a reader now sees either the old graph or the new one, never the empty gap.
