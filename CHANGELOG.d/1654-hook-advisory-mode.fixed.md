PreToolUse hook converted to advisory mode (#1654 v0.86). The hook previously emitted `continue: false` on Read of any indexed file > 4 KB with ≥ 5 symbols, which interrupted every Edit-prep workflow (Edit requires a prior Read) and every prose-doc read (markdown/planning files where `context lite=true` returns nothing useful). The fix: keep the suggestion logic identical, but emit `continue: true` + `systemMessage` so the agent sees the redirect hint without the Read being blocked. New `Decision` value `redirect_advisory` preserves telemetry — `pincher hook-stats` still counts every would-be redirect. The hook now nudges agents toward `context`/`search` without breaking iterations when those tools wouldn't actually help.
