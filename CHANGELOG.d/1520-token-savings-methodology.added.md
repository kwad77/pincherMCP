**Reproducible token-savings methodology + reproducer ([#1520 FILE-A](https://github.com/kwad77/pincher/issues/1520) v0.85):** every public token-savings claim now traces back to a falsifiable procedure. `docs/methodology/token-savings.md` documents: what "tokens saved" means (actual = response bytes ÷ 4, baseline = bytes-an-agent-would-have-read for each tool's registered `baseline_method`); the 28-tool baseline-method table mapping every registered tool to its measurement shape (`full_file_read` / `index_summary` / `none`); cheap vs exact token-accounting modes and when to use each; the reproducer script's three phases (build → measure → compare-against-baseline). `scripts/reproduce-savings.sh` builds pincher from HEAD, indexes the current repo, runs `pincher bench` at `PINCHER_TOKEN_ACCOUNTING=exact` with N=50 samples, compares against `testdata/savings-baseline.json` (committed per-release reference figures), and exits non-zero on drift beyond ±10%. Published claims in README / docs / marketing must reference the methodology doc and map to a row in the baseline file — claims that don't are removed at the next release-prep audit. Section on what we explicitly DON'T claim (universal savings, end-to-end agent-workflow savings, tokenizer parity across families) lives alongside, so reviewers can challenge the right numbers. Pairs with FILE-B (#1521 v0.86) external comparator implementations + FILE-C (#1522 v0.91) per-tool latency budget CI gate.
