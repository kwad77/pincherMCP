Fixed a data race in the supervisor tests surfaced by the new `-race` CI job (#1741 v0.89). `TestSupervisor_StatusToolReturnsResponse` and sibling supervisor tests passed a plain `bytes.Buffer` as `Supervisor.Stdout`: the supervisor's pump goroutine wrote it while the test's poll loop read `clientStdout.Bytes()`, and `bytes.Buffer` is not safe for concurrent Write + Bytes. A new `syncBuffer` (one mutex over `Write` / `Bytes` / `String`, with `Bytes` returning a copy) replaces the raw buffer across `supervisor_test.go`. This is a test-only race — a real MCP client reads the supervisor's stdout over an OS pipe, not a shared in-process buffer. Separately, the `-race` CI job's per-package timeout is raised 900s → 1800s: race instrumentation on the SQLite-heavy `internal/server` suite ran past the original budget.
