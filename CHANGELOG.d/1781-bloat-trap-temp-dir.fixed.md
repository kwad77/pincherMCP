`IsBloatTrap` now refuses the OS temp directory regardless of mode (#1781 v0.89). The bloat-trap guard refused the filesystem root and `$HOME` but not `%TMP%` / `$TMPDIR` — so a manual `pincher index` (or MCP `index` call) targeting the temp directory sailed through. On the dogfood machine that landed a 767k-symbol / 19k-file `Temp` project: the OS temp directory is never a real project root, churns constantly (any index is stale immediately), and accumulates transient build-cache / archive / test-scratch junk. The refusal is exact-match only — subdirectories of temp (e.g. a `t.TempDir()` test corpus) remain legitimate index targets. Found in the v0.89 workflow-accuracy deep dive.
