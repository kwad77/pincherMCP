The `Host conformance` CI gate is now required, not advisory (#1536 v0.91, FILE-R). The per-host canonical-workflow replay (`scripts/host-conformance.sh` against the `testdata/host-conformance/<host>/` corpus — all six supported MCP hosts: claude-code / cursor / codex / jetbrains / vscode-copilot / zed) shipped advisory in v0.87 (#1532, FILE-M) — informational, `continue-on-error: true`, never blocking. v0.91 promotes it: the `continue-on-error` flag is removed, so a failed replay now blocks the PR — including the release-prep PR, hence the release tag — and it joins the CLAUDE.md required-gates list. On a failed replay the per-host request/response transcripts upload as a `host-conformance-transcripts` artifact for offline investigation (`host-conformance.sh` honors a `HOSTCONF_OUT_DIR` env var to keep its replay scratch; the index DB is excluded from the artifact).
