**REFERENCE.md CLI subcommand sections for `supervised`, `health-check`, `stats`, `hook-stats`, `verify` ([#1509](https://github.com/kwad77/pincher/issues/1509) v0.83):** five subcommands advertised in `pincher --help`'s banner had no dedicated `### \`pincher <name>\`` section in `docs/REFERENCE.md` — they appeared only as prose mentions or as MCP-tool / HTTP-endpoint references. Pre-fix a user clicking from README → REFERENCE.md → CLI subcommands found only 11 of the 16 advertised subcommands documented, forcing them back to `pincher <cmd> --help`. Added sections following the existing shape (one-line purpose, flag/example block, cross-references to related MCP tool / HTTP endpoint / sibling subcommand). Added two parity gates so the drift can't recur: `TestReferenceMD_EveryCLISubcommandHasSection` (forward direction — every banner entry has a doc section), `TestReferenceMD_NoOrphanCLISection` (inverse — no doc section without a banner entry). `expectedCLISubcommands` in the test file is the canonical list; adding a subcommand requires bumping both it and the help banner.
