TypeScript `namespace` members now extract parented to the namespace instead of as unparented top-level symbols (#1762 v0.91). A `namespace Foo { … }` is a named container whose members should parent to it, but the natural fix — a `scopeRE` — regressed: the `regexExtractor` framework switches a scoped body to `methodRE`, and a TS namespace body holds `function` / `class` declarations, not method-syntax members, so adding the scopeRE dropped every `function` inside (#1761 reverted it, filed this). The framework gains an `nsScopeRE` — a namespace-style scope that sets `Parent` for inner symbols and emits a `Module` symbol for the namespace itself, while keeping `funcRE` for the body so the `function` members still match and stay `Kind=Function` (a namespace function is not a method). `tsRE.nsScopeRE` covers `namespace Foo` / `module Foo` (the legacy-TS alias). Existing `scopeRE` users (Rust `impl`, Swift `extension`) are untouched — `nsScopeRE` is a separate field. Found in the v0.91 issue-log triage.
