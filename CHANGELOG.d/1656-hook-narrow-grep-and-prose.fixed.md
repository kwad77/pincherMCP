PreToolUse hook narrowed to reduce noise (#1656 v0.86). Three changes after #1654 advisory-mode shipped and live dogfooding showed the hook still fired too aggressively: (1) Grep redirect converted to advisory (was still `continue: false` — broke Grep-then-Edit workflows the same way Read did before #1654). (2) Prose / planning files exempted from the Read redirect: Markdown / RST / AsciiDoc files plus anything under `.planning/`, `docs/`, `doc/`, `notes/`, or with a `.planning-` basename prefix passes through silently. `context lite=true` returns no useful symbols on prose, so the redirect was active misinformation. (3) Size floor raised from 4 KB to 16 KB — files under that are small enough that the agent reading them directly is cheaper than the hook chrome + re-issue overhead even in the advisory case. Telemetry intact via `Decision: "redirect_advisory"`.
