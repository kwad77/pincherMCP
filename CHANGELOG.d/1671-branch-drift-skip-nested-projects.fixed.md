Branch-drift advisory now skips projects nested inside another indexed project (#1671 v0.87). Same-release dogfood cascade fourth-hop: after #1665 / #1667 / #1669 trimmed obviously-false entries, the remaining "drift" on the dogfood machine was `probe` at `internal/supervisor/cmd/probe` — a real directory but nested inside the `pincher-repo` project. `git -C` against the nested path walks up to the parent repo's `.git` and reports the PARENT's branch as the nested project's, producing a phantom drift signal whenever the parent switches branches. The nested-project advisory (#1209) is the canonical surface for nested registrations; flagging "drift" here was a duplicate signal channeling users toward `pincher index <nested-path>` (which would just re-extract files already indexed by the parent). Fix: build the nested-projects set up-front from the project list and skip drift checks for any project whose normalised path lives under another indexed project's path. Both MCP and CLI doctor copies updated.
