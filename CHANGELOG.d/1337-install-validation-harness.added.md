Install-validation harness — `.github/workflows/install-validation.yml` (#1337, #668 Phase 3). On every published release (and on demand via `workflow_dispatch` against any past tag) it exercises the documented install paths on each supported platform and asserts `pincher --version` reports the released tag: 6 `direct` cells (download the release tarball/zip, checksum-verify against `SHA256SUMS`, extract, run — macOS arm64/amd64, Linux amd64/arm64, Windows amd64/arm64) and 2 `docker` cells (pull `ghcr.io/kwad77/pincher:<version>` and run it — linux amd64/arm64). Those 8 cells gate every release. The `brew` (2 cells) and `scoop` (2 cells) tiers are channel-gated to stable tags, since the Homebrew formula auto-bumps only on stable and the scoop manifest has no release auto-bump yet (gap filed #1702 — scoop cells run smoke-only meanwhile). Automates the manual cross-platform smoke from #670 §3; replaces "we'll find out when a user reports it" with a per-release gate.
