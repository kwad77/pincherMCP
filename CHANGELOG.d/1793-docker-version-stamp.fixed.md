The Docker image binary now reports the release version instead of `vdev` (#1793 v0.90). `Dockerfile`'s build ran `go build -ldflags="-s -w"` with no `-X main.version`, so the in-image `pincher --version` reported `pincherMCP vdev` — and `release.yml`'s `docker/build-push-action` step passed no `build-args` to feed one. Every published Docker image since the image first shipped carried an unstamped binary; the platform tarball builds were always stamped, so it went unnoticed until the v0.89.0 `install-validation` run — the first time its `docker` cells actually executed — failed both with `--version reported 'vdev', expected 'v0.89.0'`. The `Dockerfile` builder stage now takes an `ARG VERSION` (default `dev`) and stamps `-X main.version=${VERSION}`; `release.yml` surfaces the bare version as a step output and passes it as a `--build-arg`. v0.90.0 is the first correctly-stamped Docker image.
