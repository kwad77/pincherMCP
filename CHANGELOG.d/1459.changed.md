C# extractor promoted from confidence **0.70 → 0.85** (stable-regex tier, joining TS/TSX/Rust/Java/Swift/Kotlin). `classRE` covers `class` / `record` / `record class` / `record struct` (C# 9+ immutable types — Blazor / ASP.NET Core / minimal-API codebases lean on these heavily) / `struct` / `readonly struct` (Unity + game-dev value types); dedicated `enumRE` catches `enum` declarations; `[Attribute]` and `[Attribute(args)]` prefix tolerance on every pattern (essential for `[Serializable]`, `[ApiController]`, `[HttpGet("/x")]`, `[Required]`, `[Authorize]`, etc.); `file`-scoped access modifier (C# 11+); `partial` class / `partial` interface; `readonly` / `required` member modifiers; generic type-parameter clauses with one level of nesting (`Cache<TKey, TValue> where TKey : notnull`). Return-type token matching upgraded from `\w+` to `[\w<>?,\.\[\]]+` so generic-bearing signatures (`public async Task<IActionResult> CreateUser(User u)` — the ubiquitous ASP.NET Core minimal-API shape) match cleanly. **Framework gap also closed:** `classRE` matches now suppress `funcRE` for the same line — C# 9+ records declare primary constructors inline (`public record Person(string Name, int Age)`), which funcRE would otherwise mis-extract as a Function named Person. The class-kind keyword wins; the primary constructor is part of the type declaration, not a separate method symbol. Pre-fix typical modern C# codebases (ASP.NET Core controllers, Blazor pages, EF Core entities) were heavily under-extracted: every `record`, `struct`, `enum`, `[ApiController]`-annotated class invisible.
