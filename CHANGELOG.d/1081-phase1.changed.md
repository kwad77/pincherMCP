**MCP roots/list:** Multi-root iteration with `IsBloatTrap(hookMode=true)` clearance per root ([#1081](https://github.com/kwad77/pincher/issues/1081) Phase 1). Pre-fix `detectRoot` used `result.Roots[0]` unconditionally — if a host advertised `/` (or another bloat-trap default) at index 0 followed by the actual project at index 1, pincher set the session root to the bloat-trap path and either failed to index or polluted the DB with a catch-all walk. Post-fix `pickSessionRoot` iterates every advertised root, filters by `parseFileURI` + `IsBloatTrap(hookMode=true)`, and uses the first cleared root. `IsBloatTrap(hookMode=true)` requires at least one project marker (`.git`, `go.mod`, `package.json`, etc.) — the right strictness for roots auto-discovery since the host fires `roots/list` without an explicit user opt-in for any specific path. Single-root behavior is unchanged for hosts that advertise one root (test pins this); multi-root hosts no longer pick a bloat-trap root just because it happens to be at index 0 (test pins this); advertised ordering is preserved when multiple roots clear (test pins this). Phase 2 (background auto-index of additional cleared roots + `_meta.auto_indexed` field surfacing) rolls forward to v0.77.
