**fix(index):** Walker skips `.claude/` (Claude Code's ephemeral git-worktree convention) so parent projects no longer double-index their nested worktrees ([#1475](https://github.com/kwad77/pincher/issues/1475)). Pre-fix `.claude/worktrees/<branch>/` — Claude Code's per-feature-branch sibling working tree — got walked under the parent project's project_id, producing duplicate `file_path` rows across project IDs (the worktree was also registered as its own pincher project) and inflated symbol/edge counts on the parent (the dogfood case: parent project's `files` jumped 84 → 335 after a worktree dir appeared). `gocodewalker.ExcludeDirectory` is name-based — `.claude` is now in the list so the entire subtree is skipped before any file emits. The principled fix (detect any nested `.git` entry as a project boundary) needs gocodewalker per-dir callback support; this name-based skip is the pragmatic fix for the most common offender, and `.claude/` also holds agent settings/logs that don't belong in the code-intel graph regardless. Pinned by `TestIndex_SkipsClaudeWorktrees_1475` — corpus with `main.go` at top level + `.claude/worktrees/feature-x/main.go` + `.claude/settings.json`; only the top-level `main.go` lands in the parent's `files` table.
