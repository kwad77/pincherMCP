**Schema-migration invalidates audit + doctor surface ([#1497](https://github.com/kwad77/pincher/issues/1497) phase 1):** every schema migration in `internal/db/db.go` is now classified by what previously-extracted data it makes stale, via the new `MigrationInvalidates` type and parallel `schemaMigrationInvalidates` slice. Audit of all 32 migrations (v1 ā†’ v33): **27 classified as `invalidatesNothing`** (sessions metrics, diagnostic surfaces, metadata columns, virtual-generated columns, pure DDL drops, corpus-routing trigger updates for languages with no pre-migration rows, table rebuilds that preserve data in-place); **5 classified as `invalidatesAll`** (v18ā†’v19 `pending_edges` table introducing the persisted cross-file resolver model, v19ā†’v20 `edges.source` whose migration comment already recommended a manual `pincher index --force`, v21ā†’v22 `pending_edges.receiver_type` + `struct_fields` driving the #423/#493 receiver-type resolver, v25ā†’v26 `pending_edges.base_type` for the #565 binding pass, v30ā†’v31 `branch` column on symbols/edges/files/pending_edges that breaks any query filtering by branch). The classification is enforced by four parity tests: `TestSchemaMigrationInvalidates_ParityWithMigrations` (slice length lockstep ā€” `init()` panics at process start if a new migration is appended without a matching invalidates entry), `TestSchemaMigrationInvalidates_AllSentinelHasAll` (defensive against an accidental edit of the sentinels), `TestSchemaMigrationInvalidates_ClassificationCount` (forces a CHANGELOG/rationale update when the 27/5 split drifts), `TestSchemaMigrationInvalidates_AllEntriesAreDocumented` (rationale-keyword surface check). The new `Store.LastStartupMigrationInvalidates()` accessor returns the union across migrations applied during this process's `Open()` call, plus the version range bumped. `mcp__pincher__doctor` now surfaces this as a `startup_migrations` block ā€” `{from_version, to_version, requires_full_reindex, affected_languages?}` ā€” and attaches a warning when `requires_full_reindex=true` so users see when their last `make install` + `/mcp` reconnect genuinely needs a manual `pincher index --force` vs when it was a schema-only DDL slice with no re-extraction required. Wiring this signal into a `binaryDriftForce` gate (skip the force=true cascade when the union is `invalidatesNothing`) is the v0.80 follow-up ā€” the v0.78.1 slice ships the audit + visibility surface, not the behaviour change, to keep correctness risk bounded.
