A new test gate verifies that reader-routed `*Store` methods never reach the single-writer DB connection ([#1826](https://github.com/kwad77/pincher/issues/1826)). The reader/writer classification gate previously checked only that a method was *listed*, not that its implementation matched — the gap that let [#1824](https://github.com/kwad77/pincher/issues/1824) ship. The new gate AST-walks each reader-classified method (and every `*Store` method it transitively calls) and fails if any invokes a query/exec/begin on `s.db`. It immediately caught two more mis-routed methods — `ClosureRowCount` and `TraceViaClosure` (the trace closure fast-path) — both now routed to the reader pool.
