The incremental-resolve drift detector (#1792, [#1772](https://github.com/kwad77/pincher/issues/1772)) now snapshots the CALLS edge count at `Index()` entry, before the per-file extraction goroutines run — not only at the resolve block start. A `per_file` CALLS edge dropped during extraction and not rebound by the resolve pass was previously invisible to the detector (the resolve pass adding `resolve_pass` edges masked the deficit). A new `pincher.index.scoped_calls_net_loss` warning fires when a scoped incremental tick net-loses CALLS edges end-to-end, with a `likely_phase` field attributing the loss to extraction vs resolve. Observability only — no behavior change.
