Zig extractor now recognises `enum` / `union` / `opaque` type definitions (#1755 v0.89). `zigRE.classRE` matched only `const Name = struct`, so a Zig type defined as `enum`, `union`, `union(enum)` (tagged union), or `opaque` was not extracted at all — the type vanished from the symbol graph (and any `pub fn` declared inside it would orphan). `classRE` now matches `struct` / `union` / `opaque` with optional `packed` / `extern` layout qualifiers; a dedicated `enumRE` handles `= enum` (extracting it as `Enum` kind). No double-emission: `enum` is not in `classRE`, and a tagged `union(enum)` has `union` immediately after `=` so it matches `classRE` only. Found dogfooding the regex-tier extractor sweep.
