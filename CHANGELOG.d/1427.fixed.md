`fetch` no longer mis-extracts ordered-list or bash-comment lines as the Markdown title when the document has no real H1. Pre-fix `firstMarkdownH1` scanned every line for `# ` prefix without tracking fenced-code-block state — pincher's own README (which has zero H1s, only H2s and HTML banner divs) got stored as Document title `"1. Install"` because the Quick Start section's `\`\`\`bash` block contains `# 1. Install` as a shell comment. Now toggles an `inFence` flag on `\`\`\`` / `~~~` lines and skips any matches inside, so only real top-level H1s win. Title falls back to URL when no H1 is present anywhere outside fences (the existing behaviour, now correctly reached). New `TestFirstMarkdownH1_SkipsFencedCodeBlocks` covers bash-comment-in-fence, tilde-fence, h1-before-fence-still-wins, and multi-fence state-tracking. (#1427)
