**`onboard_module` MCP tool — Phase 4 composite #4 ([#1391](https://github.com/kwad77/pincher/issues/1391) v0.84):** new new-contributor orientation composite. Takes `directory` (relative path inside the indexed project, e.g. `internal/auth/`). Enumerates every symbol whose `file_path` is prefixed by the directory, identifies entry points + the exported surface, computes language breakdown + test-to-code ratio, and partitions CALLS edges into `external_dependencies` (the module's outbound boundary — what it depends on) + `external_consumers` (its inbound boundary — what depends on it). Returns `{scope, entry_points_local_to_scope, external_dependencies, external_consumers, module_summary}` with `scope.{directory, file_count, symbol_count}`, sorted boundary edges (capped at 50 per direction so wide-fan-out modules don't drown the agent), and `module_summary.{language_breakdown, test_to_code_ratio, exported_surface_count, entry_point_count}`. Replaces the typical 5-10 call orientation sequence (`architecture` + `search file_pattern=path/**` + `trace direction=out` × N + `context` × N). Path-prefix scan uses `LIKE 'dir/%' ESCAPE '\'` with metacharacter escaping so unsanitised inputs (`%`, `_`, `\`) don't unexpectedly match. Substring-prefix safety: `directory=core` only matches `core/`, not `core_extra/`, because the SQL prefix includes the trailing `/`. Read-only; no internal MCP round-trips per the composite contract. 9-test audit suite (positive + negative + control + cross-check) covers missing-directory rich-error, unknown-directory empty_reason, scope-count happy path, external_consumers boundary inbound, external_dependencies boundary outbound, module_summary shape, path-prefix substring safety, gate test, and pure-unit `sqlLikeEscape` truth table.
