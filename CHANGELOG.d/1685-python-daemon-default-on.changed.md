Python AST extraction now uses the persistent subprocess by default (#1685 v0.88). v0.87 (#1626) shipped the `pythonRunner` daemon opt-in behind `PINCHER_PYTHON_AST_DAEMON=1`; v0.88 flips it to default-on (opt back into per-file spawn with `PINCHER_PYTHON_AST_DAEMON=0`). The new committed bench gate — `BenchmarkPythonExtract_PerFileSpawn` vs `BenchmarkPythonExtract_Daemon` plus `TestPythonDaemon_FasterThanPerFileSpawn_1685` — measures the steady-state amortisation at ~126× (per-file spawn 34.8ms/op → daemon 0.28ms/op on the test hardware; a 12-file batch ran 419ms vs 2.7ms). The test asserts the daemon-faster-than-spawn ratio rather than an absolute threshold, so it holds cross-platform without flaking on slower process-creation. The flip is safe by construction: the daemon's JSON output is byte-identical to the one-shot path (same embedded `python_extract.py` logic, stdin loop vs argv), any daemon error falls through to per-file spawn per-call, and a hung daemon is killed by the 15s `daemonRequestTimeout` — the failure mode is "slower," never "wrong."
