`db.Open` no longer exits the process on a transient `SQLITE_BUSY` ([#1817](https://github.com/kwad77/pincher/issues/1817)). The `migrate()` step takes a write lock; another pincher mid-index (force-reindexing a large repo) can hold the writer past the 5s busy_timeout, and `#1784` only made the resulting error message friendly — the process still exited 1. `Open` now bounded-retries `migrate()` (3 attempts, 1s backoff, ~17s cap) on `SQLITE_BUSY`-class errors; a real migration error returns immediately and is never masked by the retry.
