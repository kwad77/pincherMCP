Incremental indexing now logs a `pincher.resolve.scoped_calls_net_loss` warning when a scoped resolve pass nets a CALLS-edge loss (#1772, v0.89). The #986/#1772 graph-degradation family — a scoped incremental resolve dropping CALLS edges it doesn't re-bind — has resisted static analysis: the recovery path (#457 `pending_edges` + #1678 referrer snapshot + #1629 scoped resolve) looks correct on paper, so the drift is a subtle runtime race/gap. The detector snapshots the project's CALLS edge count before and after each scoped resolve and, on a net loss, logs `before` / `after` / `lost` plus the tick's scope shape (`scope_files` / `extracted_files` / `referrer_files`) — turning silent drift into an attributable, scope-correlated event for the eventual root-cause fix. Pure observability: no behavior change; the #1774 / #1786 self-heal still restores the graph.
