`guide` no longer carries comparison-filler words into its templated `search` recommendation (#1682 v0.87). Dogfood-found: the task "add memory-pressure backoff to the indexer's resolve pass **like** the watcher has" produced a `search query="indexers resolve pass like"` recommendation — the filler token "like" dragged into the FTS5 query, where it only matched via the AND→OR fallthrough rather than naming anything real. `taskHintFromString`'s longest-non-stopword-run heuristic was selecting a run that ended in "like" over the cleaner subject run. Eight comparison/filler words — `like`, `similar`, `such`, `than`, `too`, `also`, `just`, `same` — added to the stopword set so they break runs instead of joining them. The exact-token filter means filler substrings inside real identifiers (`likelihood`) are unaffected.
