Go extractor's CALLS pass no longer emits phantom CALLS edges for invocations through locally-shadowed names (parameters, receiver, or body-local `name := func(){}` / `var name = func(){}` bindings). Sibling fix to #1423 but for the CALLS path rather than READS. Pre-fix repro on pincher-repo: `internal/server/server.go::classifyTaskShape` has `contains := func(needles ...string) bool {...}` then calls `contains("dead code", ...)` — every call site emitted a CALLS edge with ToName=`contains` that the resolver bound to `cmd/pinch/init_hook.go::main.contains`, a Function in a totally different package that Go's scope rules wouldn't reach. New `goLocalCallableNames(d)` pre-scans the FuncDecl for shadowed callable names (params + receiver + body FuncLit assignments via `:=` and `var`); `extractGoCalls` skips bare-name callees that match the shadow set. Selector calls (`obj.Method`, `pkg.Fn`) are unaffected — the resolver's receiver-type / receiver-method paths handle disambiguation there. New `go_callable_shadow_test.go` covers closure shadow, var-form shadow, function-typed param, unshadowed regression guard, selector-not-filtered cross-check, and the dogfood `classifyTaskShape` repro shape. Existing `TestExtractGoReads_ParamShadow_RealCallStillEmitsCALLS` from #1423 renamed to `_NoCALLSOrREADSToShadowedName` to reflect the now-symmetric suppression — when a param shadows a Function name, BOTH the READS and CALLS for that name suppress.
