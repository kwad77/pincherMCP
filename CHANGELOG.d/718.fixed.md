**Two silent-failure gaps in `symbols` and `query` ([#712](https://github.com/kwad77/pincher/issues/712) follow-up).** The original Group A failure-as-pedagogy sweep missed `symbols`' two arg-rejection paths — empty `ids` array and over-cap batch both still returned a bare text error. They now return the rich `_meta.next_steps` envelope: an empty batch points at `search` / `query` (the tools that produce IDs), an over-cap batch explains the paging split. Separately, `query` silently dropped the legacy `cypher` alias when a caller passed *both* `pinchql` and `cypher` — same silent-ignore class as #473's unknown-property bug. It now runs `pinchql` and surfaces a `_meta.warnings` entry naming which one ran.
