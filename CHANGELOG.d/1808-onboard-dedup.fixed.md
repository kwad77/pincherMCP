`onboard_module` now reports a SET of distinct boundary dependencies instead of a multiset of call-sites ([#1808](https://github.com/kwad77/pincher/issues/1808)). `external_dependencies` / `external_consumers` appended every boundary `CALLS` edge with no `(from_id,to_id)` dedup, so a function that calls a boundary symbol N times flooded the list with N identical rows — a live probe saw one pair surface 6× — and the `maxEdgesPerList = 50` cap then truncated genuine distinct dependencies in favour of the duplicates. The new `dedupBoundaryEdges` helper collapses repeats before the sort + cap.
