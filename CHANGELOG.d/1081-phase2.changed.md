**MCP roots/list Phase 2 ([#1081](https://github.com/kwad77/pincher/issues/1081) v0.78):** background auto-index of every advertised root that clears `IsBloatTrap(hookMode=true)`, beyond the session-root pick. Phase 1 (v0.76) iterated the roots list to pick a non-bloat-trap session root; Phase 2 fires `Index()` in a goroutine for every remaining cleared root so the agent can `search project=<name>` against any advertised workspace without an explicit `index path=...` call first. The audit trail lives in `Server.autoIndexedRoots` (each entry transitions `"indexing"` → `"indexed"` / `"error: ..."` as the goroutine completes). Server-internal for v0.78 — Phase 3 (v0.81+) surfaces it via `_meta.auto_indexed` on the first tool call after init, which needs the `_meta` envelope contract sequencing that ships alongside the composite-tool work. The selection logic `clearedSessionRoots` is broken out from `pickSessionRoot` so each side has its own pure helper; Phase 1's behavior (single-cleared-root → session root, advertised ordering preserved) is unchanged. Audit shape pinned by `TestClearedSessionRoots_ReturnsAllCleared` (positive) + `TestClearedSessionRoots_AllTrapsReturnsEmpty` (negative) + `TestStartAutoIndex_RecordsIndexingThenIndexed` (status transition) + `TestStartAutoIndex_PreservesAdvertisedOrder` (cross-check).
