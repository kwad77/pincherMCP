Kotlin extractor promoted from confidence **0.70 → 0.85** (stable-regex tier, joining TS/TSX/Rust/Java/Swift). `classRE` covers `class` / `data class` / `sealed class` / `inline class` / `value class` / `annotation class` / `inner class` / `companion object` / `object` (Kotlin's singleton pattern, previously invisible); dedicated `enumRE` catches `enum class` declarations (the two-word keyword form, distinct from standalone `enum` in other languages); dedicated `interfaceRE` catches `interface`, `fun interface` (SAM, Kotlin 1.4+), and `sealed interface`; `funcRE` tolerates `@Annotation` and `@Annotation(args)` prefixes (essential for `@Composable`, `@JvmStatic`, `@Throws`, `@OptIn`, etc.), `inline` / `tailrec` / `infix` / `operator` / `override` / `abstract` / `open` / `external` / `expect` / `actual` modifiers, and generic function type parameters between `fun` and the name (`fun <T : Comparable<T>> max()`). Extension-function syntax `fun Type.method()` from v0.67 #1183 stays covered. Pre-fix typical Kotlin codebases (Android with Compose, Ktor services, Kotlin Multiplatform projects) were heavily under-extracted: every `object` singleton invisible, every `interface` missing, every `enum class` missing, every `@Composable` function missing. **Framework also bumped:** `enumRE` matches now set `currentClass` so enum methods scope as `Method` with the enum as `Parent`, mirroring the #819 `interfaceRE` shape — Swift enum methods (`localizedDescription()` inside a `NetworkError` enum) also benefit retroactively. Generic type-parameter clauses tolerate ONE level of nesting (`<T : Comparable<T>>`) via the shared `<[^<>]*(?:<[^<>]*>[^<>]*)*>` shape. (#1457)
