`investigate_failure` no longer ranks Go SIGSEGV panic-header words or hyphenated repo-path segments as crash suspects ([#1806](https://github.com/kwad77/pincher/issues/1806)). A textbook `invalid memory address or nil pointer dereference [signal SIGSEGV: segmentation violation ...]` panic leaked `violation` (matched a workflow-lint Class) and the repo name `pincher` (from a `pincher-repo` checkout dir, matched the Homebrew formula) as false-positive suspects — scoring high enough to evict the panic's real top frame from the capped rank. `stopwordFrames` gains the full SIGSEGV header vocabulary and `parseStackFrames` drops any token immediately followed by `-` (a hyphenated path segment, never an identifier). Extends the #1787 fix.
