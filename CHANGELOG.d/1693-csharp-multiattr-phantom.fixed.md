C# regex extractor no longer captures the last attribute on a multi-attribute field/property line as a phantom Method (#1693 v0.89). Sixth fix from the #1389 cross-language false-positive sweep. funcRE's return-type token class `[\w<>?,\.\[\]]+` included `[` and `]` (needed mid-token for `byte[]` array return types). On a line like `[SerializeField, Range(0f, 1f)] private float phase2At = 0.66f` the attribute-skip group backtracks to zero, `[SerializeField,` is then eaten as a return-type token, and the last attribute's name (`Range`) is captured as a phantom Method — Unity codebases hit this on every `[SerializeField, Range(...)]` field, so repeated phantoms collided on qualified_name. The token class now requires a return-type token to START with a word char (a real type never begins with `[`; `byte[]`/`Task<T>`/`int?`/`System.IO.Stream` all still parse, and `[` `]` remain legal mid-token). Reindex impact: fools-gold-pirate 13→9 qualified_name_collision rows.
