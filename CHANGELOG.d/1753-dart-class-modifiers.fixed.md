Dart extractor now recognises Dart 3 class modifiers (#1753 v0.89). `dartRE.classRE` matched only `(abstract)? class Name`, so a class declared with `sealed` / `final` / `base` / `interface` / `mixin` (Dart 3's class-modifier keywords) was not extracted at all — and because the regex framework's classRE-tracked `currentClass` never saw the type, every method inside it degraded from `Method` (parented) to a free `Function` with empty `Parent`. `search kind:Class` missed the type; `trace` / `dead_code` / `audit_unused` saw orphan Functions. `classRE`'s modifier group now tolerates any sequence of `abstract` / `base` / `interface` / `final` / `sealed` / `mixin` before `class`. No double-emission with `interfaceRE` — its name capture requires `[A-Z]…` and the keyword `class` is lowercase, so `interface class Foo` / `mixin class Foo` match `classRE` only. Found dogfooding the regex-tier extractor sweep.
