The HTTP dashboard gains a **Doctor tab** ([#1812](https://github.com/kwad77/pincher/pull/1812)) — index health at a glance, mirroring `pincher doctor` and driven by the existing `/v1/doctor` endpoint. It shows the environment line (binary / schema / DB size / WAL), the doctor advisories verbatim (DB bloat, ghost/nested projects), a per-project health table that flags schema-drifted rows and offers a one-click force [Re-index] per project, and extraction failures grouped by reason. Previously this information was only reachable from the `pincher doctor` CLI; a user whose agent was getting stale answers had no in-dashboard way to see why.
