`changes` (and every `trace` / `context_for_task` call) no longer serializes behind indexer writes ([#1824](https://github.com/kwad77/pincher/issues/1824)). The recursive-CTE trace and five session/all-time metric reads were classified reader-routed but ran their pure-SELECT queries on `s.db` — the single-writer connection (`SetMaxOpenConns(1)`) — so each query queued behind the watcher's index writes. `changes scope=all` fires one trace per changed symbol; a doc-heavy diff (378 symbols) took 145-348s. All six are now routed to the reader pool (`s.ro`); WAL readers don't block on the writer.
