**Capability-advertisement doc parity test ([#672](https://github.com/kwad77/pincher/issues/672) workstream 4 v0.79 hardening):** new `TestCapability_EveryAdvertisedTagDocumented` in `internal/server/capability_doc_parity_test.go` parses the canonical capability table in `docs/integrations/meta-envelope-contract.md` and asserts every tag in `computeCapabilities()` has a documented row (and every doc row maps to either a default-advertised or known-conditional tag). Fixed inline (`mcp_logging` row added) and gated structurally so the next capability addition can't ship undocumented. Mirrors the existing `TestCapability_EveryAdvertisedTagHasRuntimeProbe` gate's role for runtime parity.
