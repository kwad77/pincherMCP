**`why_empty` MCP tool + empty_reason catalog ([#1391](https://github.com/kwad77/pincher/issues/1391) + [#1534 FILE-O](https://github.com/kwad77/pincher/issues/1534) v0.85):** new `why_empty` composite — Phase 4 #5, the final composite — converts a previously-returned `_meta.empty_reason` value into a structured recovery action. Stateless catalog lookup (no DB query, no project scope) backed by `docs/empty-reasons.md` (also new in this release per FILE-O), which enumerates all 12 `EmptyReason*` constants with: when each fires, recovery action, machine-readable `recovery_steps[{tool, args, why}]` array, and a catalog anchor. The composite replaces "agent receives empty → reads catalog doc → tries each recovery probe one at a time" with one round-trip — passing `prior_empty_reason="no_results_in_corpus"` returns the matching catalog entry with the recovery_steps already populated. Unknown reasons return a rich error listing every known value for typo recovery. Pairing gates: `TestWhyEmpty_CatalogCoversEveryConstant` (every constant has a catalog entry) and `TestWhyEmpty_NoUnknownCatalogEntries` (no catalog entry refers to a missing constant) — adding a new `EmptyReason*` requires bumping the catalog AND the doc AND the test suite in lockstep. 8-test audit suite (positive + negative + control + cross-check) covers missing-input rich-error, known-reason catalog hit, unknown-reason rich-error with typo-recovery listing, both pairing gates, recovery-step shape contract, stateless-no-DB invariant, and gate test. **Phase 4 composite suite complete:** investigate_failure (v0.81) + plan_change (v0.82) + audit_unused (v0.83) + onboard_module (v0.84) + why_empty (v0.85) = 5 of 5 shipped per `docs/integrations/composite-tool-roadmap.md`.
