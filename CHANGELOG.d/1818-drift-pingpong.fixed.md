Two pincher processes of different binary versions watching one project no longer ping-pong force-reindexes ([#1818](https://github.com/kwad77/pincher/issues/1818)). Binary-drift detection force-reindexed on ANY `binary_version` mismatch, so a release MCP and a dev `pincher web` dashboard each force-reindexed and re-stamped on every watcher tick — re-triggering the other forever, leaving the index permanently mid-wipe. The drift force now fires only when the running binary is strictly NEWER than the project's stamp (a genuine upgrade), and `UpsertProject` gained the version-level downgrade guard so an older binary can no longer stomp the stamp on a same-schema pass.
