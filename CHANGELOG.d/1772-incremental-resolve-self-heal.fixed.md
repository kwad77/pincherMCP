The watcher now self-heals the graph after incremental indexing (#1772 v0.89). An incremental-tick resolve scopes its work to the changed files (the #1629 fast path); under conditions still being pinned, that can leave the graph with fewer CALLS / IMPORTS / READS edges than a `force` reindex would produce — edges silently dropped from files outside the tick's scope. The graph degrades over a working session (a `--force` reindex of pincher-repo restored ~2,000 edges, ~14%), and every graph-shaped tool degrades with it: `trace` misses hops, `context` returns partial callees, `dead_code` / `audit_unused` over-report. The watcher now runs a **resolve-only self-heal pass** once a project settles into idle after a burst of incremental reindexes: `Indexer.ResolveOnly` re-resolves the project's full persisted `pending_edges` pool project-wide — identical to a force-reindex's resolve phase — but skips re-extraction entirely (the ~82% cost per the #1611 measurement), so it is cheap. It runs once per active→idle settle, never on a steady-state idle project, and is a safe idempotent no-op on an already-whole graph. This is a mechanism-agnostic mitigation — it restores dropped edges regardless of the exact incremental-resolve drop cause; the root cause stays open under #1772 for a precise fix.
