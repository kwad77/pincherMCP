New `antigravity-mcp` init target — registers pincher as an MCP server in Google Antigravity (#1770 v0.89). The `antigravity` target (#1765) writes a rules file that tells Antigravity's agent *about* pincher; it does not make pincher's tools *callable*. `pincher init --target=antigravity-mcp` registers pincher as an MCP server in `~/.gemini/antigravity/mcp_config.json` — the file Antigravity's "Manage MCP Servers → View raw config" opens — so the Antigravity agent can invoke `search` / `context` / `trace` / the rest directly. It writes the standard MCP `{"mcpServers": {"pincher": {"command", "args": ["supervised"], "env"}}}` shape, merging into any existing config (preserving the user's other servers) and refusing to clobber malformed JSON — the `vscode-mcp` writer pattern. Antigravity's MCP config is global (per-user), so the target is `AlwaysGlobal` like `codex`: the MCP `init` tool surfaces it as `skipped_always_global`; write it via the `pincher init --target=antigravity-mcp` CLI. Completes the Antigravity integration — `antigravity` (rules) + `antigravity-mcp` (MCP-server registration), the same split as `vscode` / `vscode-mcp`.
