The Scoop manifest now auto-bumps on every stable release (#1702 v0.91). `release.yml` bumped the Homebrew formula and re-tagged the Docker image after a release, but `packaging/scoop/pincher.json` was static — pinned at v0.60.0, so the documented `scoop install <raw manifest URL>` path shipped a binary many minors stale. A new `scoop` job mirrors the `homebrew` job: stable-channel-gated (minor divisible by 10), it fetches the release `SHA256SUMS`, patches the manifest's `version`, the two Windows artifact URLs, the two SHA256 hashes, and the version-stamped `bin` exe names, then commits to `master`. It runs `needs: homebrew` so the two `push origin master` commits serialise rather than racing non-fast-forward. With the manifest now tracking the release tag, the `install-validation` harness's `scoop` cells can be promoted from smoke-only to strict version assertion. Found in the v0.91 issue-log triage.
