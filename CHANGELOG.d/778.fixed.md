**`pincher init` injected an LF policy block into a CRLF file, leaving mixed line endings.** On a Windows repo whose `CLAUDE.md` (or other target file) uses CRLF, `init` kept the original content's `\r\n` but wrote the `<!-- pincher:start -->...<!-- pincher:end -->` block with bare `\n` — and `strings.TrimRight(existing, "\n")` on the append path left a dangling lone `\r`. `MergePolicyBlockBare` now normalizes `existing` to LF for the merge logic, remembers whether the original used CRLF, and restores that convention on output — every CRLF-input path produces uniform CRLF, LF-origin files stay LF. Found during v0.56 dogfooding.
