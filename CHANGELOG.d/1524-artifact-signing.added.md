**Release artifact signing scaffolding (FILE-E, #1524).** `.github/workflows/release.yml` now signs every `pincher-*.tar.gz`, `pincher-*.zip`, and `SHA256SUMS` with cosign when the `COSIGN_PRIVATE_KEY` repository secret is configured; signatures attach to the GitHub release as `*.sig` sidecars. Until the secret is configured the workflow stays green and logs a `::warning::` line — unsigned releases still ship checksums. Maintainer setup and end-user verification documented in [docs/security/release-signing.md](docs/security/release-signing.md). Closes the third leg of the v1.0 supply-chain triplet (threat model #1553 + govulncheck #1552 + signing).
