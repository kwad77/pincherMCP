`index` no longer misdiagnoses a healthy no-change re-index as a vendor-only directory (#1773 v0.89). `diagnoseEmptyIndex` checked the `all_files_blocked` case (`Files==0 && Blocked>0`) before the `incremental_no_change` case, so a fully-indexed project's benign no-change incremental tick — which hash-skips its source files (`Skipped>0`) *and* blocks its lockfiles / minified bundles (`Blocked>0`) — matched `all_files_blocked` first and reported `empty_reason: all_files_blocked` with the hint "expected for vendor-only or build-artifact-only directories," telling the agent a project with thousands of indexed symbols had no sources. The `all_files_blocked` branch now also requires `Skipped==0` — blocked must be the *only* reason nothing was processed; a genuine vendor-only directory (all files blocked, none hash-skipped) still diagnoses correctly, while a healthy hash-skip pass now correctly reports `incremental_no_change`.
