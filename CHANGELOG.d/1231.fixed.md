Root-cause diagnosis for #1231 (silent symbol loss in shared-DB Watch path) committed as a SKIPPED regression test (`internal/db/cross_project_id_collision_test.go::TestSymbol_CrossProjectIDCollision_BothMustSurvive`). The `symbols` table has `id TEXT PRIMARY KEY` without `project_id`; `MakeSymbolID` returns `"{file_path}::{qualified_name}#{kind}"` with no project scope. Two projects with the same relative file path containing the same symbol produce identical IDs, and `INSERT OR REPLACE` silently flips the row's `project_id` to the latest writer. This matches the live-bug shape (pincher-repo's server.go shows only 8 of 75 Methods because sniffer — an older mirror — also indexed the same relative path and clobbered the 67 pre-existing rows). The structural fix (composite PK or project-scoped `MakeSymbolID`) tracks in a follow-up PR; this regression test is the GREEN gate for it. ([#1231](https://github.com/kwad77/pincher/issues/1231))
