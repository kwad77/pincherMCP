`guide` no longer presents a docstring query as the answer to every unrecognized audit task (#1759 v0.89). `inferAuditPinchQL` keyword-matched complexity / line-count / untested audits, and its `default` case bolted `n.docstring IS NULL` onto the query for *every other* audit-shape task — so "find every function that writes to the database without going through the reader pool" and "find every HTTP handler that lacks authentication checks" both got a query about missing docstrings, with a `why` claiming "structural audit — pinchQL filters on docstring/is_exported directly" as if docstring were the task. An agent running the recommendation got undocumented functions, unrelated to its intent. `inferAuditPinchQL` now has an explicit case for docstring / undocumented / comment tasks (which keeps the docstring query); the `default` drops the arbitrary docstring filter, emits the neutral candidate set (exported non-test Go Functions + Methods), and uses an honest `why` — guide states it could not compile the specific predicate into a `WHERE` clause and the agent should add their own predicate (`schema` lists the available properties) or filter the results manually. Found dogfooding the `guide` tool.
