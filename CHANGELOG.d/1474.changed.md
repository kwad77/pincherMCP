**Force-index hot path — fewer SQL parses + skip redundant `git rev-parse` ([#1474](https://github.com/kwad77/pincher/issues/1474)):** CPU profiling of the force-reindex path showed ~42% of cumulative time in SQLite statement parsing and ~42% in `os/exec.Cmd.Start` subprocess work. Two targeted fixes drop both. **(1) `DeleteSymbolsForFile` now uses subquery-based bulk deletes** instead of looping over symbol IDs and issuing 3 individual `tx.Exec` per symbol; the per-file cleanup is now a fixed 4 statements regardless of symbol count (was `1 + 3N`). On a 700-file project with ~10 symbols/file that's ~21,000 SQL parses → 2,800, a 7.5× reduction. Profile confirms `_sqlite3RunParser` / `_sqlite3Prepare` / `_sqlite3Parser` cumulative share dropped from ~21% each → ~3.5% each (6× drop in parser pressure). **(2) `detectGitBranchCached(force=true)` skips the subprocess when `.git/HEAD` content is byte-for-byte unchanged** since the cached entry. Pre-fix, `force=true` unconditionally spawned `git rev-parse --abbrev-ref HEAD` (+ a `--short HEAD` fallback for detached cases); rapid-fire force calls from the post-schema-migration auto-restart storm (the [#1474](https://github.com/kwad77/pincher/issues/1474) repro fires 6 parallel force=true within ~2s) re-spawned git for every call. Post-fix, an unchanged `.git/HEAD` (the file `git checkout` rewrites atomically) is sufficient proof the branch hasn't moved, and the subprocess is skipped. A real `git checkout` rewrites `.git/HEAD` so `feat/x` is picked up immediately on the next force pass — pinned by `TestDetectGitBranchCached_ForceReDetectsOnHEADChange` and the existing `TestIndex_BranchSwitchUpdatesCurrentBranch`. **Wall-clock impact on pincher-repo force pass:** 6.77s → 6.27s median (3 runs each, ~7% faster); compounds to ~3 seconds across the user's 6-project parallel reindex storm. New tests: `TestDetectGitBranchCached_ForceSkipsSubprocessOnUnchangedHEAD` (positive — cachedAt does not advance on second force call), `TestDetectGitBranchCached_ForceReDetectsOnHEADChange` (correctness — HEAD content change invalidates cache), `TestDetectGitBranchCached_NonGitDirReadsBlank` (fail-open — non-git dirs fall back to subprocess path unchanged), `TestReadGitHEAD_TrimsTrailingNewline`.
